## 1. 项目概述

### 1.1 项目目标
构建一个迷你"五险一金"计算器Web应用，核心功能是：
- 根据员工工资数据和城市社保标准（佛山）
- 计算公司为每位员工应缴纳的社保公积金费用
- 清晰展示计算结果

### 1.2 核心约束
- **仅支持佛山市**的社保标准
- 清空重新生成计算结果（每次计算前清空results表）
- 使用配置文件管理Supabase连接信息

## 3. 数据库设计

## 4. 核心业务逻辑

### 4.1 计算流程

```
┌─────────────────────────────────────────────────────────────┐
│                     五险一金计算流程                          │
└─────────────────────────────────────────────────────────────┘

1. 数据获取
   └─ 从 salaries 表读取所有工资记录

2. 分组计算
   └─ 按 employee_name 分组
   └─ 计算每位员工的年度月平均工资
       公式：SUM(salary_amount) / COUNT(DISTINCT month)

3. 获取城市标准
   └─ 从 cities 表获取佛山的 year、base_min、base_max、rate

4. 确定缴费基数
   └─ 对于每位员工：
       IF avg_salary < base_min THEN contribution_base = base_min
       ELSE IF avg_salary > base_max THEN contribution_base = base_max
       ELSE contribution_base = avg_salary

5. 计算公司缴费
   └─ company_fee = contribution_base * rate

6. 存储结果
   └─ 清空 results 表
   └─ 插入所有员工的计算结果
```

### 4.2 数据示例

**输入数据 (salaries)**:
| employee_name | month | salary_amount |
|---------------|-------|---------------|
| 张三 | 202401 | 8000 |
| 张三 | 202402 | 9000 |
| 李四 | 202401 | 1500 |
| 李四 | 202402 | 2000 |

**城市标准 (cities - 佛山)**:
| base_min | base_max | rate |
|----------|----------|------|
| 1900 | 31056 | 0.15 |

**输出结果 (results)**:
| employee_name | avg_salary | contribution_base | company_fee |
|---------------|------------|-------------------|-------------|
| 张三 | 8500.00 | 8500.00 | 1275.00 |
| 李四 | 1750.00 | 1900.00 | 285.00 |

## 8. 重要注意事项

### 8.1 配置文件管理
- Supabase连接信息通过 `config/supabase.js` 管理
- 敏感信息使用环境变量，不直接硬编码
- `.env.local` 不提交到版本控制

### 8.2 数据验证
- 上传的Excel数据需要验证字段完整性
- 检查必填字段是否存在
- 验证数据类型是否正确

### 8.3 错误处理
- 所有可能失败的操作都需要错误处理
- 向用户返回清晰的错误提示信息
- API返回统一的错误格式

### 8.4 性能考虑
- 大批量数据上传使用批量插入
- 结果页考虑分页（如果数据量大）
- 添加适当的加载状态提示
